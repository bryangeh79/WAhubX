# WAhubX · 产品介绍书

> 最新更新: **2026-04-26** · 含 D11 WA Status 自动化 + 客服接管 CDP screencast + 双引擎双号架构

## 一、产品定位

**WAhubX** 是面向马来西亚本土及东南亚华人圈的 **WhatsApp 多账号自动化运营 SaaS 平台**。专为中小企业、电商卖家、本地服务商、外贸团队打造,让你用 10 个、30 个乃至 50 个 WhatsApp 账号同时开展养号、营销、客服、广告投放,所有操作集中在一个桌面端 App 里完成。

我们的口号:**养号一次到位,投放自动跑,客户不丢单**。

### 一句话讲清楚 WAhubX 是什么

把 50 个销售员塞进一台电脑 · 24 小时不睡觉 · 自动养号 · 自动跑广告 · 自动接客户 · 全程数据可回看 · **数据全在你电脑里 · 平台一个字都看不到**。

---

## 二、我们解决什么问题

1. **多号手动管理爆炸**:一个员工盯 5 个号都手忙脚乱,账号多了容易漏回、错回、甚至被 WhatsApp 风控封号。
2. **客户回复跟不上**:广告投出去 300 人,回 30 个,人工一条条回 3 小时起跳,回复慢了就流失。
3. **群发即死号**:新号刚注册就群发,1-2 天内被封是家常便饭。
4. **没有数据**:发了多少、收了多少、成本多少、哪个号效果好、哪个文案转化高?全靠经验猜。
5. **合规压力**:WhatsApp 不断加强反机器人,纯暴力工具越来越危险。

## 三、核心功能

### 3.1 账号槽位管理 (Slot Management)

- 支持 **Basic 10 号、Pro 30 号、Enterprise 50 号** 三档套餐,按槽位数收费。
- 每个槽位 = 一个独立的 WhatsApp 账号,带独立代理 IP、独立指纹、独立操作历史。
- 扫码绑定手机号后,系统自动建立会话并持续维护在线。
- 断线自动重连(线性退避 30 秒起,最多 5 次),闲置超过 30 分钟自动释放操作锁。
- 每个号的 SIM 卡信息、运营商、绑定国家、WhatsApp 昵称全部可在后台查看。

### 3.2 养号系统 (Warmup)

号买来不能直接用,必须先"养"。WAhubX 内置 **7 天 4 阶段成熟号计划**(也可选 14 天稳健模式):

- **Phase 0 · Incubate (Day 1-2)**:新号保活,只接收消息,不主动行为,严守 WA 服务端 72h 硬冷却。
- **Phase 1 · Preheat (Day 3-4)**:开始与内部号互发消息,累积正常聊天记录。
- **Phase 2 · Activate (Day 5-6)**:扩展到外部低风险联系人,小幅发消息,开始发 WA Status (动态)。
- **Phase 3 · Mature (Day 7+)**:跑剧本对话,完全模拟真人社交,**解锁广告投放**。

系统自动排班,每天按照时段窗口(默认 10-12 / 14-17 / 19-22)执行任务,夜间 02:00-06:00 绝不发送,最大程度规避风控。养够 7 天的号标记为"成熟号"(Mature Slot),可开始接广告投放。

**养号模板** (一键启用):
- `v1_7day` — 7 天加速版 · 适合新 SIM 进入业务期
- `v1_14day_full` — 14 天稳健版 · 适合零关联老号
- 群养号 plan: 4-8 号一起跑, 互聊互看, 关联自然

**单号实测**: 严格按 7 天 plan 跑后, 直接进 Phase 3 跑广告, 7 天首日 200 条 / 30 天 5000+ 条无封号 (内部 pilot 数据)。

### 3.3 广告投放 (Ad Campaign)

完整的 4 步投放向导:

1. **时间安排**:立即发送 / 指定时间 / 每天定时 / 每周指定星期。
2. **目标客户群**:支持从联系人挑选、CSV/Excel 批量上传、粘贴号码三种方式导入,自动去重、规范化手机号为 E.164 格式。
3. **文案与开场白**:一个投放最多选 2 条文案,可配合开场白。系统支持 AI 自动生成文案变体池(每条最多 30 个变体),真发时每个目标随机选一个变体,大幅降低文字重复率。
4. **执行方式**:"系统智能"自动挑选成熟号 / "自定义槽位"指定号码。

**节流三档** (保守/平衡/投放):

- 保守档 (默认): 每号每天上限 20 条 · 时段 10-12/14-17/19-22 · 消息间隔 40-120 秒随机。
- 平衡档: 每号每天 30 条 · 间隔 30-90 秒。
- 投放档: 每号每天 40 条 · 时段 10-12/19-22 · 间隔 20-60 秒。

**承载安全算法**:系统实时计算成熟号数 × 每日上限 × 投放天数 = 总容量,与目标人数对比。≥100% 绿灯可启动、70-100% 黄灯提示、<70% 红灯禁止,避免号池吃不下。

**真人打字模拟**:发每条消息前自动模拟"正在输入..."状态,按文字长度计时(每字 80-150ms,下限 1.5 秒、上限 8 秒),让客户感受不到机器。

**投放结果报告** (完成后自动生成):

- 总览: 发送成功率、回复率、执行时长
- 账号表现: 每号分配、成功、失败、成功率
- 文案表现: A/B 对比哪条更好
- 时段分布: 柱状图显示按小时发送量
- 失败分类: 按 WhatsApp 返回的错误码分组
- 一键导出 CSV · 一键复制投放

### 3.4 智能客服 (Intelligent Reply)

**这是 WAhubX 的王牌功能**。客户收到广告后回复消息,系统自动处理:

**3 种模式**:

- **关闭**:100% 人工处理
- **智能** (推荐):系统按 FAQ 匹配优先 → 不命中用 AI 回答 → AI 不确定转人工
- **草稿**:AI 生成回复建议,人工点发送才发

**设置简单到离谱**:只需 3 步

1. 上传公司/产品介绍 PDF 或 Word 文档
2. 选择"智能"模式
3. 写一句业务目标(如:让客户留下电话 · 引导预约 demo)

完事!后续客户一切回复系统全自动处理。

**内部智能机制** (藏起来的自动化):

- 接收客户消息后等 8 秒聚合(避免客户连发三条消息被回三次)
- 24 小时最多自动回复 3 次,30 分钟内不重复回
- 客户问"多少钱/见面/预约/投诉"等关键词立即转人工
- AI 回复自带 Guardrail:保留原文的电话、邮箱、网址不改;过滤"100%/保证/绝对承诺"等风险词;价格数字自动替换为"请联系顾问"
- 进入人工接管后 AI 彻底闭嘴,不插话
- RAG 检索置信度:≥0.75 放心答 / 0.45-0.75 保守答 + 人工兜底 / <0.45 直接转人工

**AI 生成 FAQ**:

租户上传完产品文档,系统用 DeepSeek 自动生成 30 条常见问答初稿,默认"待审核"状态,租户审核通过后才启用。每条 FAQ 有启用/草稿/停用三种状态,还能导出 CSV、批量上传 Excel。

### 3.5 人工接管 (Human Takeover)

AI 搞不定的对话自动推到"人工接管" tab,销售人员一键接管:

- 对话顶部显示客户来源(哪个广告投放带来的)
- AI 历史判断("高意向"/"询价"/"投诉"等)
- 相关广告摘要
- 支持发文本、图片、语音、文件
- 30 分钟闲置自动释放锁,让号继续跑养号任务
- 28 分钟预警,避免话说一半断了

### 3.6 客户群管理

- 建立多个客户群 (比如"618 客户"/"VIP 客户"/"询价未成交")
- 每个群支持批量操作、克隆、导出 CSV
- **自动坏号标记**:发送失败的号码自动标记为"坏号",下次发同一群自动跳过,杜绝重复浪费
- **引用统计**:看每个客户群被多少个投放引用过
- **好友占比**:群内多少是好友、多少是陌生人

### 3.7 多 AI 供应商支持

租户可自由选用:

- **DeepSeek** (推荐中英文/马来语场景,成本最低)
- **OpenAI GPT** (英文创作最强)
- **Google Gemini** (免费额度大)
- **Anthropic Claude** (长文本分析优秀)

每个租户自己填 API Key,用量独立结算。平台提供兜底 AI (DeepSeek),新租户零配置也能先体验 FAQ 生成等基础功能。

### 3.8 WA Status (动态/Story) 自动化 🆕

WhatsApp 的 Status (类似 IG Stories) 是高曝光低成本的获客渠道, 但手动发繁琐. WAhubX 全自动化:

- **自动发文字 status** — 池里随机抽文案, 每号每天 1-2 条, 时段错开
- **自动发图片/视频 status** — 从 persona 素材池或内置兜底素材池抽 (M7 已 seed)
- **自动浏览他人 status** — 进 viewer 停留 3-8 秒, 模拟真人阅读, 留 view 痕给作者 (你出现在他 viewer list 里 = 软引流)
- **自动点赞 status** — 每天硬上限 5 条, 防风控
- **自动改"关于"/签名** — 模拟真人会变签名, 增加号活跃信号
- **Phase 门控**: status 类任务从 Phase 2 (Day 5+) 才解锁, 新号绝不暴露

**应用场景**: 客服号每天发产品图 / 营销号每天发专属 promo / 静默"挂在线"号每天点赞 5 条互动 — **不发广告, 但每天有动作 = 号一直被 WA 视为活号**。

### 3.9 客服接管中心 · 系统内嵌 WA Web 🆕

接管真客服对话不用切窗口, **直接在 WAhubX 控制台里看到客服号的真 WA Web 画面**:

- **CDP screencast 嵌入** — 后端通过 Chrome DevTools Protocol 把客服号 Chromium 画面以 5 fps 推到前端 canvas
- **真鼠标 + 真键盘转发** — 你点 canvas / 打字, 反向通过 CDP 注入到客服号 Chromium, 等同于真人在用 WA Web
- **客户档案只读面板** — 接管页底部显示该客服号的所有对话历史 + 高保真消息存档 (可导出 TXT / 客户名单 CSV)
- **接管期间 AI 闭嘴** — 自动暂停所有 AI 自动回复 / 任务调度
- **30 分钟自动释放锁** + 28 分钟预警, 不卡死号

### 3.10 角色分离 · 客服号 + 广告号双号架构 🆕

每个 slot 可选一个角色:

- **🛎️ 客服号** (`customer_service`): always-on · 24/7 在线 · inbound 路由到 AI 自动回复 / 接管页. 每个租户**只能有 1 个**客服号 (D11-3 唯一约束)
- **📢 广告号** (`broadcast`): 跑广告投放 / 养号任务 · inbound 不持久化 (回复直接丢)

**为什么这么设计**: 广告号被回复就停下处理 = 慢 + 风控. 双号路由后 → 广告号专心跑量, 客服号专心接客 · **互不干扰**, 且 WA 服务端看到不同号有不同行为模式 = 反指纹关联。

### 3.11 双引擎架构 · Chromium + Baileys 🆕

业内独家 · 同一套系统支持两种 WhatsApp 接入方式, 按场景自动选:

- **Chromium 真 WA Web 通道** (主推荐 · D6-D12 落地):
  - 跑真 puppeteer-core + 真 Chrome
  - WA 服务端看到的就是真用户在用 WA Web
  - 反检测三件套 (stealth-inject / human-behavior / 独立设备指纹)
  - **每号独立 Chromium 子进程** + 独立 user-data-dir + 独立 TLS 栈
  - 单号崩溃不拖垮其他号
- **Baileys 协议通道** (兼容老路径 / 备用):
  - 直接走 noise-protocol 协议
  - 适合极轻量场景或 Chromium 不可用时

**租户无感切换**: facade 层自动按 `RUNTIME_MODE` 路由, 业务代码不变。

### 3.12 反检测三件套

WA 在协议层不断加强反 bot, WAhubX 三层防护:

1. **Stealth Inject** (`stealth-inject.ts`): 注入 JS 伪装 navigator / WebGL / canvas / audio fingerprint, 让 WA Web 探不到 puppeteer 痕
2. **人类行为模拟** (`human-behavior.ts`): 打字 80-150ms/字 + 思考停顿 + 鼠标抖动, 不再是"瞬间打完"
3. **独立指纹** (`fingerprint.json` per slot): UA / 时区 / 屏幕尺寸 / 设备型号 全独立, 每号一份, **不漂**

### 3.13 安全与隐私

- **本地桌面部署**:所有数据在租户自己电脑上,平台不碰业务数据
- **License 激活**:VPS 只做授权验证,不做数据中转
- **机器指纹绑定**:每个 License 绑定一台设备,防盗用
- **加密存储**:AI API Key 以 AES-256-GCM 加密存 DB
- **数据库备份**:每天凌晨 03:00 自动备份,保留 7 天
- **绑号纪律**: 24h 静默期 + 同租户绑号 cooldown 600s · 防 WA 协同登录反作弊
- **JWT 双 token + 5 次失败锁 15 分钟**
- **全局 unhandledRejection / uncaughtException 拦截**: 单个号文件系统异常不拖垮整个 backend

## 四、产品亮点 · 为什么选 WAhubX

### 4.1 唯一一家做本地桌面版的 WhatsApp 自动化工具

市面上 90% 竞品是云端 SaaS,需要把你的 WhatsApp session 上传到他们服务器。一旦对方被黑、跑路、或配合监管,你的客户数据、聊天记录全泄露。WAhubX 全部数据存本地,平台完全接触不到。

### 4.2 7/14 天成熟号机制 · 行业独家

市面上"WhatsApp 批量工具"大多是"绑号就群发",7 天内封号率 60% 以上。WAhubX 强制 7 天 (急用) 或 14 天 (稳健) 养号才准投放,封号率降到行业最低(内部 pilot <5%)。Phase 0 严守 WA 服务端 72h 硬冷却, 这条规则是行业封号大户没遵守的。

### 4.3 完整数据闭环

从广告发出 → 客户回复 → AI 自动回 → 意向判断 → 转人工 → 成交/流失,**全流程数据留痕**。你能回放任何一个客户的完整旅程,分析漏斗在哪一步流失。

### 4.4 成本可控

- 平台订阅费透明(RM 200/月起)
- 租户自己的 AI API Key · 用多少付多少
- 平台兜底 AI 月限 20 次(足够大部分租户启动期免费使用)
- 无额外"成功案例费""客户激活费"等套路

### 4.5 中文优先 + 马来西亚本土化

- 全中文 UI
- 手机号默认 MY 规则(0 开头自动加 60 前缀)
- 时区默认 Asia/Kuala_Lumpur
- 节流窗口按马来西亚人生活节奏设计

## 五·三、合规与免责 (重要 · 必读)

### 5·3.1 数据合规
- **马来西亚 PDPA** (个人资料保护法 2010) 友好: 数据全部存租户本地 PostgreSQL, 平台不碰业务数据, 无跨境数据流动
- **GDPR 友好**: 同上, 本地化存储天然符合 GDPR Article 28 数据处理人要求

## 六、常见疑问

**Q: 会被 WhatsApp 封号吗?**
A: 严格按照系统节流跑,封号率远低于业内。我们的 14 天养号 + 真人打字模拟 + 分时段发送 + 同 IP 互斥等多层保护,是目前最合规的方案。

**Q: 数据会外泄吗?**
A: 不会。WAhubX 是本地部署,聊天记录、客户数据全部存你自己电脑上的 PostgreSQL 数据库,平台方完全接触不到。

**Q: 需要技术背景吗?**
A: 完全不需要。首次装完跟着引导扫码绑号,点几下就能跑起来。我们有视频教程 + 微信群答疑。

**Q: 支持 macOS / Linux 吗?**
A: V1 只支持 Windows 10/11。macOS 计划 V2。

**Q: 需要多少台设备?**
A: 一台 License 绑一台机器。如果要在两个地方使用,需要买第二个 License。

**Q: 数据可以从别的工具导入吗?**
A: 支持。客户群支持从 Excel / CSV 批量导入,wa_contact 可从现有手机端同步。

**Q: 我的号已经在用了 (现役号), 能直接绑吗?**
A: 可以但**不推荐**直接绑现役号. 现役号关联的设备/IP 历史很复杂, 绑入系统后可能直接 440 死循环踢出. **推荐流程**: 用新 SIM 绑号 → 跑 7 天养号 → 把现役号客户慢慢迁过来.

**Q: 绑号失败 / 一直 "正在同步连接" 怎么办?**
A: 通常是: ① 老号已被 WA 标记 (换新 SIM); ② 代理 IP 不干净 (换 residential proxy); ③ 同租户最近 10 分钟绑过其他号 (等冷却). 控制台 → 槽位卡 → 💡 诊断 看具体原因.

**Q: 客服号被人接管时, 还能跑自动回复吗?**
A: 不会. 接管期间 AI 自动暂停, 释放锁后恢复. 保证客户不会同时收到"AI + 人"两条回复.

**Q: WA Status 自动发布会被风控吗?**
A: 风险极低. 我们硬上限 (1-2 条/天 文字 / 5 条/天 浏览 / 5 条/天 点赞) + Phase 2 才解锁. 只比真人活跃度略低, 不暴露.

**Q: 平台收我的客户数据用来训练 AI 吗?**
A: **完全不会**. WAhubX 是本地部署, 业务数据 100% 在你电脑的 PostgreSQL 里. 我们没办法收 (技术上做不到), 也没必要收 (不是我们的商业模式).

**Q: 一个号能跑多个 WhatsApp 账号吗?**
A: 不行 · 每个 slot 对应一个 WA 号. 但**一个 WAhubX 实例可以管 50 个号**, 这才是产品价值.

**Q: 有 macOS / Linux 版本吗?**
A: V1 只支持 Windows 10/11. macOS 排在 V2 (2026-Q4). Linux 暂无计划 (需求少).

**Q: 如果 WhatsApp 大改协议怎么办?**
A: 双引擎架构是核心保险. Chromium 路径跟 WA Web 走, 只要 WA Web 能开, 我们就能跑. Baileys 路径是备用, 即使 Chromium 路径暂时挂, 也能切回去. 我们承诺 V1 期内任何 WA 协议变化, **48h 内出热修补丁**.
